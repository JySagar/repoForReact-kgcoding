-> Including Bootstrap

2. Components : Pre-styled elements like buttons and navbars.
3. Customizable : Modify default styles as needed.
4. Cross-Browser : Consistent look across browsers.

-> Fragments

1. What ? : It allows grouping of elements without extra DOM nodes.
2. Why ? : i> Return multiple elements without a wrapping parent. <ii> Cleaner DOM and consistent styling.
3. How ? : Two syntaxes : i> <React.Fragment>...</React.Fragment>. <ii> <>...</>

-------------------------------------------------------

-> Map Method

1. Purpose : Render lists from array data.
2. JSX Elements : Transform array items into JSX.
3. Inline Rendering : Directly inside JSX.
eg: {items.map(item => <li key={item.id}>{item.name}</li>)}
4. Key Prep : Assign unique key for optimized re-renders.
eg: <div key={item.id}>{item.name}</div>

-------------------------------------------------------

-> Conditional Rendering

1. Conditional Rendering : 
i> Displaying content based on certain conditions.
ii> Allows for dynamic user interaces.

2. Methods :
i> If-else statements : Choose between two blocks of content.
ii> Ternary operators : Quick way to choose between two options.
iii> Logical operators : Useful for rendering content when a condition is true.

3. Benefits :
i> Enhances user experience.
ii> Reduces unnecessary rendering.
iii> Makes apps more interactive and responsive.
